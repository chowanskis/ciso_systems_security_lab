# ciso_systems_security_lab

This repository contains practical examples of C programming and SQLite usage
from a security perspective.

The goal is to understand how vulnerabilities are introduced, exploited, and mitigated
in low-level applications.

Topics include:
- Memory safety issues (buffer overflows, use-after-free)
- Input validation and sanitization
- Secure file and process handling
- SQLite security (injection, misuse, data exposure)
- Defensive coding patterns
- Secure architecture mindset for C-based systems

This project is designed for security engineers and CISO-level professionals
who want to better understand application-layer risks at the code level.
